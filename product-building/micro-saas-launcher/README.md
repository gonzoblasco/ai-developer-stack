# Micro-SaaS Launcher

Guía estratégica y playbooks de lanzamiento rápido para Micro-SaaS y MVPs comerciales bajo la filosofía Indie Hacker: entregas rápidas, validación temprana y crecimiento rentable.

## Resumen

Esta habilidad ayuda a fundadores solitarios (solopreneurs) y pequeños equipos a conceptualizar, estructurar, monetizar y lanzar productos de software en semanas en lugar de meses. Reúne directrices para evitar errores de distribución, diseñar estrategias de precios psicológicos y ejecutar planes de lanzamiento efectivos en canales como Product Hunt o Hacker News.

## Inicio Rápido

Para validar una idea y planificar un lanzamiento rápido:

1. **Framework de Validación**: Pregúntate si el problema existe, si la gente pagaría por la solución (pre-ventas o competidores existentes) y si puedes construir el MVP en 2 semanas.
2. **Definir el Stack de Lanzamiento**: Utiliza herramientas preconfiguradas que aceleren el desarrollo (Next.js para frontend/API, Supabase/Clerk para Auth y Base de Datos, y Stripe/Lemon Squeezy para cobros).
3. **Planificar el MVP Speed Run**: Centrarse en una única funcionalidad clave (Core Feature) y posponer optimizaciones complejas de escalabilidad.

## Qué Incluye

- **Fases del MVP Speed Run**: Cronograma detallado de 2 semanas (Semana 1: Auth, UI y Core Feature; Semana 2: Landing, Emails, Términos Legales y Soft Launch).
- **Psicología de Precios (Pricing)**: Tiers recomendados, fórmulas para calcular el precio en base al costo alternativo, y diagnóstico de errores en la página de precios.
- **Canales de Distribución y Tracción**: Playbook para lanzamientos en Product Hunt, Hacker News, Reddit y estrategias post-lanzamiento.
- **Solución de Problemas de Negocio (Sharp Edges)**: Estrategias para corregir problemas de distribución (Distribution First), mercados que no quieren pagar y retención de usuarios (Churn).

## Conceptos Clave

- **Distribución Primero**: No programar un software sin definir antes cómo se llegará a los clientes finales.
- **Validación con Dinero**: Es preferible conseguir intenciones de pago tempranas (preventas) que signups gratuitos que nunca se actualicen a planes de pago.

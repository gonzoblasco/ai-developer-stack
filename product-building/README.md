# 🏗️ Playbook de Construcción de Productos

Skills orientadas a **construir productos completos** desde cero usando plataformas low-code/no-code + AI.

A diferencia de `strategy-architecture/` (planificación abstracta) o `tools-automation/` (scripts y workflows), esta categoría está enfocada en el ciclo completo de vida de un producto: desde el primer prompt hasta el MVP en producción.

## Filosofía

1. **Producto primero**: Los prompts están pensados para generar valor de negocio, no solo código.
2. **Sistema de Diseño antes que features**: Siempre establecer el sistema de diseño antes de construir.
3. **Edge Functions son el backend**: En el contexto de Lovable Cloud, las Edge Functions reemplazan a n8n, servidores propios y cualquier otra infraestructura.
4. **Iteración incremental**: Cada sesión con Lovable debería terminar con algo funcional y visible.

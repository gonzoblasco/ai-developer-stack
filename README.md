## 📋 Workflows

El valor del stack no está en la cantidad de skills, está en **cómo se combinan** para resolver problemas reales. Cada workflow es un camino probado que conecta skills en un flujo cohesivo.

### 🧠 [De la Idea al MVP](workflows/idea-to-mvp.md)
Discovery → Arquitectura → Scaffold → Build → QA → Deploy

### ♿ [Auditoría y Fix de Accesibilidad](workflows/accessibility-audit.md)
Auditar → Diagnosticar → Autofix → Verificar → Reportar

### 🧹 [Limpieza de Vibe Code](workflows/vibe-code-cleanup.md)
Auditar → Limpiar → Refactor → Testear → Endurecer

### 📐 [Arquitectura y Decisiones Técnicas](workflows/architecture.md)
Diseño → ADRs → Patrones → Seguridad → Documentación

### 🧪 [Calidad y Testing](workflows/quality-testing.md)
TDD → Unit → Integration → E2E → CI/CD → Review

### 🤖 [AI Agent Setup](workflows/ai-agent-setup.md)
Diseño → Tools → Prompts → RAG → MCP → Producción

## 🚀 Uso

Estas skills están diseñadas para ser consumidas por Agentes de IA (Cursor, Windsurf, Claude Code, Codex, OpenClaw) o usadas como referencia por desarrolladores humanos. Cada carpeta contiene un `SKILL.md` (la lógica/prompt central) y frecuentemente archivos de soporte.

Empezá por los [workflows](workflows/) — elegí el que resuelva tu problema actual y seguí el flujo.

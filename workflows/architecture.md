# 📐 Workflow: Arquitectura y Decisiones Técnicas

## Objetivo

Diseñar la arquitectura de un sistema documentando cada decisión técnica con ADRs, diagramas C4 y patrones comprobados.

## Skills involucradas

| Fase | Skill | Propósito |
|------|-------|-----------|
| 1. Diseño | `senior-architect` | Diseño de sistemas, trade-offs, diagramas |
| 2. Documentación | `architecture` | ADRs (Architectural Decision Records) |
| 3. Patrones | `architecture-patterns` | Hexagonal, Clean, DDD |
| 4. Multi-tenancy | `saas-multi-tenant` | Separación por tenant |
| 5. Orquestación | `design-orchestration` | Enrutamiento de workflows de diseño |
| 6. Seguridad | `security-audit` | Threat modeling, RLS, guards |

## Flujo

```
Requerimientos → Diseño → ADRs → Patrones → Seguridad → Documentación
```

## Outputs

- Diagramas C4 de arquitectura
- ADRs de cada decisión técnica
- Diseño con patrones definidos (Hexagonal, Clean, DDD)
- Auditoría de seguridad

# 🧹 Workflow: Limpieza de Vibe Code

## Objetivo

Tomar código generado por IA (vibe coding) y llevarlo a un estado producible: sin deuda técnica, con tipos, tests y estructura sólida.

## Skills involucradas

| Fase | Skill | Propósito |
|------|-------|-----------|
| 1. Diagnóstico | `vibe-code-auditor` | Detectar patrones de deuda técnica |
| 2. Limpieza | `vibe-code-cleanup` | Endurecimiento de producción |
| 3. Refactor | `refactor` | Refactorización incremental segura |
| 4. Frontend | `lovable-cleanup` | Limpieza de exports Lovable Cloud |
| 5. Calidad | `testing-patterns` / `typescript-expert` | Tests y tipado estricto |
| 6. Estándares | `clean-code` / `software-architecture` | Convenciones y estilo |

## Flujo

```
Auditar → Limpiar → Refactor → Testear → Endurecer
```

## Outputs

- Código sin deuda técnica detectable
- Tests pasando
- TypeScript estricto sin `any`
- Estructura alineada con convenciones del proyecto

# 🧠 Workflow: De la Idea al MVP

## Objetivo

Llevar una idea desde la chispa inicial hasta un MVP funcional en producción, pasando por validación, arquitectura, scaffolding y QA.

## Skills involucradas

| Fase | Skill | Propósito |
|------|-------|-----------|
| 1. Ideación | `brainstorming` | Técnicas de ideación estructurada |
| 2. Discovery | `product-manager` / `product-manager-toolkit` | Validación de mercado, RICE, PRD |
| 3. Definición | `not-a-vibe-coder` | Planificación multi-documento |
| 4. Arquitectura | `senior-architect` / `architecture` | Diseño de sistemas, ADRs |
| 5. Scaffolding | `app-builder` | Generación de estructura base |
| 6. Construcción | `lovable-saas-builder` / `micro-saas-launcher` | MVP con Lovable + Supabase |
| 7. Planning | `plan-writing` / `new-task` | Planes de implementación |
| 8. QA | `vibecode-production-qa-validator` | Validación de launch-readiness |
| 9. Release | `pr` / `close-epic` | PR, review y cierre de epic |

## Flujo

```
Idea → Brainstorming → PRD → ADRs → Scaffold → Build → QA → Deploy
```

## Outputs

- PRD validado
- ADRs de decisiones técnicas
- MVP funcional en producción
- Epic cerrado con CHANGELOG actualizado

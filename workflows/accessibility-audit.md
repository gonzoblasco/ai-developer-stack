# ♿ Workflow: Auditoría y Fix de Accesibilidad

## Objetivo

Auditar, diagnosticar y corregir problemas de accesibilidad WCAG 2.2 en aplicaciones web, garantizando cumplimiento AA.

## Skills involucradas

| Fase | Skill | Propósito |
|------|-------|-----------|
| 1. Auditoría inicial | `accessibility-audit` | Escaneo completo WCAG, teclado, screen readers |
| 2. Diagnóstico | `wcag-audit-patterns` | Guía de remediación accionable |
| 3. Autofix | `accesslint-audit` | Corrección automática de issues |
| 4. Verificación | `ui-a11y` | Verificación de componentes StyleSeed |
| 5. Documentación | `documentation-expert` | Reporte de accesibilidad |

## Flujo

```
Auditar → Diagnosticar → Autofix → Verificar → Reportar
```

## Outputs

- Reporte de auditoría WCAG 2.2
- Issues corregidos (automática y manualmente)
- Componentes verificados
- Documentación de accesibilidad
